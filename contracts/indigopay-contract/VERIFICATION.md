# Formal Verification of IndigoPay Contract

This document outlines the formal verification approach for the IndigoPay Soroban contract using **Kani**. The goal is to mathematically prove four critical safety invariants:

1. **Global Total Consistency** – The contract‑wide `global_total_raised` always equals the sum of `project.total_raised` for all projects.
2. **Badge Tier Monotonicity** – A project's badge tier never decreases over time.
3. **Overflow Safety** – No arithmetic operation can overflow, preventing panic attacks.
4. **Pause Guard Enforcement** – The `require_not_paused` guard is called before any state‑mutating function.

## Running Verification Locally
```sh
# From the repository root
cd contracts/indigopay-contract/verification/kani
cargo kani --release
```
The command will compile the verification crate and execute the Kani harnesses. A successful run exits with status 0 and produces a report in `target/kani`.

## CI Integration
A nightly GitHub Actions workflow (`verification.yml`) runs the same command on a clean environment. It uploads the `target/kani` directory as an artifact named `kani-verification-report`. Failures are reported but do not block PRs.

## Adding New Invariants
To extend verification, add additional `#[proof]` functions in `src/lib.rs` following the existing examples. Use `kani::assume` to constrain inputs and `any()` to generate nondeterministic values.

## Known Limitations
- Kani currently verifies only Rust‑level safety (overflow, panics). Logical properties beyond what is expressed in the harnesses may need a more expressive tool such as Creusot.
- The verification job is nightly to avoid slowing PR checks.
